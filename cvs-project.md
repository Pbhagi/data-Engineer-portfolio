# Healthcare KPI Analytics & Reporting System

## 📌 Overview
Designed and developed an end-to-end analytics and reporting system for healthcare claims, pharmacy, and clinical data. The project focuses on transforming large-scale raw healthcare datasets into actionable insights through dashboards and automated pipelines.

## 🎯 Objectives
- Track and monitor 15+ KPIs across claims and compliance
- Enable data-driven decision-making for stakeholders
- Improve data quality and reporting reliability
- Automate reporting workflows

## 🛠️ Tech Stack
- SQL (BigQuery, Snowflake)
- Python
- Looker, Tableau, Power BI
- Apache Airflow
- GCP (Dataplex, Data Catalog)
- Great Expectations

## 📊 Key Features
- Built dashboards tracking KPIs like claims volume, payer trends, and compliance metrics
- Automated data pipelines processing 3+ TB of healthcare data
- Implemented data validation using Great Expectations
- Applied row-level security for regulatory reporting
- Monitored Airflow workflows for SLA adherence

## 🔄 Data Pipeline Flow
1. Data ingestion from healthcare systems
2. Data transformation using SQL
3. Data quality checks using Great Expectations
4. Storage in BigQuery/Snowflake
5. Visualization using BI tools

## 📈 Impact
- Enabled reporting for 50+ stakeholders
- Improved data accuracy and reliability
- Reduced manual reporting efforts
